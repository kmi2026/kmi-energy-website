# KMI Energy 独立域名映射与配置完全指南
*(本手册包含为“非技术/域名管理同事”准备的解析向导，以及为“程序员/开发者”准备的收尾待办事项)*

---

## 👩‍💻 第一阶段：将阿里云域名绑定到 Vercel (非代码人员操作)
本部分面向负责在阿里云管理和购买域名的同事。请按照以下步骤，将我们建好的海外 Vercel 高速节点与您的企业顶级域名（例如 `www.yourcompany.com`）对接。

### 步骤 1：从 Vercel 系统获取服务器“接头口令”
1. **登录系统**：前往 [vercel.com](https://vercel.com/)，使用主项目负责人的 Github 账号联合授权登录。
2. **进入项目**：在仪表盘列表中找到并点击进入项目卡片：**`kmi-energy-website`**。
3. **域名设置**：点击顶端菜单栏最右侧的 **Settings (项目设置)** 面板，随后在网页左侧垂直边栏中找到 **Domains (域名)** 菜单。
4. **添加域名**：在输入框内原封不动地输入您在阿里云买的带 `www` 的域名（例如：`www.yourdomain.com`），然后按回车或点击最右边的  **Add (添加)** 按钮。
5. **获取秘密通道值**：此时您会看见页面弹出显示“配置非法（Invalid Configuration）”红字。这非常正常！稍微往下看，Vercel 会呈现给您一条专属的 **CNAME 记录值**（那是一段类似于 `cname.vercel-dns.com` 的英文代码），请鼠标右键极其精准地将这串英文字符**复制**下来。

### 步骤 2：在阿里云 (Alibaba Cloud) 后台将通道打通
1. 回到**阿里云系统**，登录您的账号并使用搜索栏进入 **“云解析 DNS”** 产品控制面板。
2. 找到您属于公司那个域名，并且点击右侧出现的蓝字按钮 **“解析设置”**。
3. 在解析列表左上侧点击蓝色大按钮 **“添加记录”**，并准确地按下面这样填空：
   * **记录类型 (Type)：** 在下拉菜单果断选择 `CNAME`
   * **主机记录 (Name)：** 填入三个字母 `www` 
   * **记录值 (Value)：** 粘贴您刚刚从 Vercel 那边成功复制的那串英文密码（例如 `cname.vercel-dns.com`）
4. 点击“确认保存”。
5. 阿里云一般需要 **5~15分钟** 让全球互联网都认识这个新规则。只要时间一到，您再去刷新 Vercel 的那个域名的配置页面，红色感叹号就会变成**绿色的校验钩子**，并宣告您企业网站成功全自动申请并下发了安全小绿锁 (Https) 证书。

## 👨‍💻 第二阶段：核心授权对接待办清单 (后续跟进 TODO)
*(本部分为主程序员或运维开发者阅读的**必须执行**项。严重警告：当第一阶段成功，使用新域名预览不出错时。如若忽略此处两项收尾配置，您的 CMS 后台将会遭遇全局瘫痪故障，因为防跨域机制会拦截所有的非法域名！)*

- [ ] **待办核心项 1: [Github 端] 切换云端应用的验签白名单 (OAuth URLs)**
    1. 使用管理员权限 Github 账号。前往 `Settings` -> 左下角 `Developer Settings` -> `OAuth Apps` 之中，点进去我们用于 CMS 登入认证创建的那个应用（如 “KMI CMS”）。
    2. 无情覆盖旧的框栏内容：将 **Homepage URL** 变更为：`https://www.您的阿里云域名.com`
    3. 特级注意防呆拼写：将 **Authorization callback URL** 变更为：`https://www.您的阿里云域名.com/api/callback`
    *(此行核心机制是为了通知 Github，在您的这个高贵企业域名的网页上发起的登录不是诈骗与黑客，并允许它通行回调)*

- [ ] **待办核心项 2: [代码底层] 撤销临时防丢失硬编码配置**
    1. 在源码系统或直接点开 Github 仓库内名叫 `admin/config.yml` 的核心配置文件。
    2. 在它的开头第 5 行，存在一条旧版的系统补丁设定： `base_url: https://kmi-energy-website.vercel.app/api`
    3. **将其全面根除并替换成刚才绑定生效的顶级域名 URL（请一定要带 `/api` 尾巴）：** 
       更改为 `base_url: https://www.您的阿里云域名.com/api`
    4. 执行 Commit 变动并推送至 `main` 分支。Vercel 会在收到推送的刹那立即抓取源码并在 1 分钟内重构新平台，大功告成！
