## 👩‍💻 第一阶段：将阿里云域名绑定到系统 (非代码人员负责)
本部分面向负责在阿里云管理和购买域名的同事。请登录阿里云后台，进行以下及其简单的域名解析操作：

### 阿里“云解析 DNS” 操作步骤
1. 回到**阿里云系统**，登录您的账号并使用搜索栏进入 **“云解析 DNS”** 产品控制面板。
2. 找到本公司购买的域名 `kmienergy.com`，并且点击右侧出现的蓝字按钮 **“解析设置”**。
3. 在解析列表左上侧点击蓝色大按钮 **“添加记录”**，并准确地按下面这样填空：
   * **记录类型 (Type)：** 在下拉菜单果断选择 `CNAME`
   * **主机记录 (Name)：** 填入单主机头（例如 `www`，或依据具体路由规划）。
   * **记录值 (Value)：** **完整粘贴并且不能多加任何空格复制下面这串专属指令密码**：
     `995eecf2ad83d81b.vercel-dns-017.com`
     *(如果后面有那个小点 `.` 也可以保留原样不动)*
4. 点击“确认保存”。
5. 阿里云一般需要 **5~15分钟** 让全球互联网都认识这个新规则。只要时间一到，您的企业网站在全球范围就能通过企业域名成功访问了。

*(辛苦了！域名侧已竣工，请通知前端开发同事进行第二阶段配置，以保证网站的 CMS 内容管理台顺利激活。)*

## 👨‍💻 第二阶段：核心授权对接待办清单 (程序员/运维开发者 TODO)
*(本部分为主程序员或运维开发者阅读的**必须执行**项。严重警告：当第一阶段阿里云域名绑定成功且能正常看网页后。如若忽略此处的配套修改跟进，您的 CMS 内容后台将会遭遇全局瘫痪故障，因为防跨域安全机制会立刻拦截所有的旧域名登录！)*

- [ ] **待办核心项 1: [Github 端] 切换云端应用的验签白名单 (OAuth URLs)**
    1. 使用管理员权限 Github 账号。前往 `Settings` -> 左下角 `Developer Settings` -> `OAuth Apps` 之中，点进去我们用于 CMS 登入认证创建的那个应用（“KMI CMS”）。
    2. 无情覆盖旧的框栏内容：将 **Homepage URL** 变更为正式上线的地址：`https://kmienergy.com` 
    3. 特级注意防呆拼写：将 **Authorization callback URL** 变更为含接口后缀的形式：`https://kmienergy.com/api/callback`
    *(备注说明：如果贵司最终敲定了大家都必须要带 `www.` 这个词头来访问官网，那么您在这里所有的链接和配置里请同样硬性必须加入 `www.`)*

- [ ] **待办核心项 2: [代码底层] 设定为正式域名**
    1. 在电脑的源码代码或是直接点开 Github 仓库内名叫 `admin/config.yml` 的核心结构文件。
    2. 在它的开头第 5 行，原本为了兼容旧网络它长这样：`base_url: https://kmi-energy-website.vercel.app/api`
    3. **立刻将它全面替换成您最终敲定的企业域名加上 `/api` 尾巴的严格 URL 格式：** 
       更改为形如：`base_url: https://kmienergy.com/api` (同样是否带 www 视您第一步规划而定)
    4. 执行 Git Commit 变动并推送提交您的代码至 `main` 分支发布。Vercel 极其灵敏，会在收到推送的不到半分钟内重构新平台，至此 CMS 后台全面融入专有域名！
